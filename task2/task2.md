## Задание 2. Разработка и заполнение проверочного листа для бизнес-систем

| №  | Вопрос                                                                                                           | Да/Нет | Комментарий                                                                                                                                                                                                 |
|----|------------------------------------------------------------------------------------------------------------------|--------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Управление доступом**                                                                                                                                                                                                          |
| 1  | Используется ли многофакторная аутентификация (MFA)                                                              | Нет    | В системе есть чувствительные данные: личная информация, документы о сделках, платёжные поручения                                                                                                          |
| 2  | Используются ли средства централизованного контроля доступа?                                                     | Да     | Единое хранилище учётных записей                                                                                                                                                                           |
| 3  | Используются ли системы аутентификации пользователей для доступа к системам?                                     | Да     | Едина точка входа позволяет контролировать процесс аутентификации                                                                                                                                          |
| 4  | Ведётся ли журналирование всех операций доступа к системам и данным?                                             | Нет    | Система предоставляет доступ к разным данным разным группам пользователей, хотелось бы видеть журнал доступа к данным                                                                                       |
| 5  | Созданы ли замкнутые контуры для подсистем, работающих с чувствительными данными?                               | Нет    | Разграничение доступа к данным по ролям и привилегиям необходимо для контроля потока данных                                                                                                                 |
| 6  | Реализован ли принцип минимальных привилегий при назначении ролей и прав доступа?                               | Нет    | Система вообще не содержит механизма ролей, привилегий. Аналитикам BI нужны отдельные от бизнес-аналитика права и привилегии                                                                               |
| **Безопасность данных**                                                                                                                                                                                                          |
| 1  | Разработана ли политика классификации данных, и актуализируется ли она в соответствии с действующим законодательством? | Нет    | В текущей реализации все данные лежат в одной куче. Нет шифрования и обезличивания                                                                                                                          |
| 2  | Используется ли шифрование на уровне базы данных, файловой системы и при передаче данных по сети?               | Нет    | Шифрование необходимо для защиты секретных данных                                                                                                                                                           |
| 3  | Проводится ли регулярное резервное копирование данных?                                                          | Нет    | Иметь копию данных в защищённом хранилище критически важно                                                                                                                                                  |
| 4  | Применяются ли средства или процессы анонимизации данных?                                                       | Нет    | Обезличенное хранение данных необходимо для соответствия 152-ФЗ                                                                                                                                             |
| 5  | Настроены ли VPN и шифрование для безопасного удалённого доступа к корпоративным ресурсам?                     | Нет    | Бизнес-аналитик и BI должны подключаться через VPN                                                                                                                                                         |
| **Защита инфраструктуры**                                                                                                                                                                                                        |
| 1  | Используется ли межсетевой экран для ограничения входящего и исходящего трафика?                               | Да     | Внутренняя система должна быть изолирована от внешней                                                                                                                                                      |
| 2  | Настроен ли мониторинг сетевого трафика на предмет аномалий и возможных атак, например, DDoS?                 | Нет    | Это важный аспект                                                                                                                                                                                          |
| 3  | Осуществляется ли фильтрация трафика между сетевыми сегментами?                                                | Нет    | Есть несколько внутренних компонент, нужно следить за тем, чем они обмениваются                                                                                                                            |
| **Управление инцидентами**                                                                                                                                                                                                       |
| 1  | Используется ли система мониторинга безопасности (SIEM) для корреляции событий и анализа аномалий в режиме реального времени? | Нет    | Для своевременного реагирования угрозы                                                                                                                                                                     |
